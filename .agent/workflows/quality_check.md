# 品質チェック・セルフレビュー手順 (/quality_check)

実装タスクが完了し、ユーザーに報告する直前にこのチェックリストを実行してください。
すべての項目を確認し、問題があれば修正してから完了報告を行ってください。

## 1. 📱 モバイル・レスポンシブ確認
- [ ] **画面幅縮小テスト**: ブラウザ幅をスマホサイズ（約375px）まで縮めてレイアウト崩れがないか確認しましたか？
- [ ] **タップ領域**: ボタンやリンクはスマホ指で押しやすいサイズ・配置になっていますか？（特に削除ボタン等の危険な操作）
- [ ] **見切れ防止**: 長いテキスト（名前、メモ）が画面外にはみ出していませんか？(`break-word` / `truncate` / 折り返し設定)

## 2. 🛡️ データおよびロジックの堅牢性
- [ ] **重複ガード**: 一意性が求められるデータ（予約時間など）に対して、サーバーサイドでの重複チェックを実装しましたか？
- [ ] **整合性維持**: データを削除または更新した際、関連するデータ（来院回数、リレーション）の整合性は保たれていますか？
- [ ] **サーバー検証**: クライアント側のチェックだけでなく、Server Action内でもZod等による入力値検証を行っていますか？

## 3. ⚠️ エラーハンドリングとUX
- [ ] **エラーフィードバック**: エラー発生時、ユーザーに分かりやすいメッセージ（トーストやアラート）が表示されますか？
- [ ] **ロード状態**: 処理中（Pending）にボタンを無効化し、二重送信を防いでいますか？
- [ ] **フォームリセット**: 送信成功後、フォームの内容は適切にクリア（または保持）されていますか？

## 4. 🧹 コード品質
- [ ] **クリーンアップ**: デバッグ用の `console.log` や、不要なコメントアウトコードは削除しましたか？
- [ ] **型定義**: `any` 型を避け、適切な型定義を行っていますか？
