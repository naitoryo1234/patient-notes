## 🏗️ Base Kit vs カスタム機能

**重要な設計方針**: 機能を「Base Kit（標準）」と「カスタム（有料/上位）」に明確に区分し、Base Kitをシンプルに保つことで汎用性を確保する。

### ✅ Base Kit（標準搭載）

| 機能 | 状態 | 備考 |
|------|------|------|
| 患者管理（CRUD） | ✅ 完了 | 検索、タグ付け、基本情報管理 |
| カルテ記録（SOAP形式） | ✅ 完了 | AI取込モード含む |
| 予約管理（リスト表示） | ✅ 完了 | 重複チェック、担当者割当 |
| ダッシュボード | ✅ 完了 | 本日の来院、スマートディレクトリ |
| スタッフ管理 | ✅ 完了 | 基本的な追加・無効化 |
| バックアップ（エクスポート） | ✅ 完了 | SQLiteファイルのダウンロード |
| ラベル管理（変数化） | 🚧 一部完了 | 主要コンポーネントのみ適用 |

### 💎 カスタム機能（将来 / 有料）

> 詳細は [IDEAS.md](management/IDEAS.md) を参照

| 機能 | 優先度 | 状態 | 検討事項 |
|------|--------|------|----------|
| 予約カレンダービュー | ⭐⭐⭐ | 保留 | ON/OFF切替設計を検討 |
| 繰り返し予約 | ⭐⭐⭐ | 未着手 | 定期患者向け |
| 業種別プリセット | ⭐⭐ | 議論中 | 医療/サロン/シンプル等 |
| LINE連携 | ⭐ | アイデア段階 | API課金発生 |
| 音声入力 | ⭐ | ブランチ保留 | AI整形との統合待ち |
