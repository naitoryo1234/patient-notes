# 運用プロトコル (Operational Protocols)

## 📌 アイデア・提案の保存プロセス (Idea Archiving Process)

「汎用的な土台（Base Kit）」としてのシンプルさを維持しつつ、将来的な拡張性や特定のユースケースに対応するためのアイデアを適切に管理するためのルールです。

### 1. 提案の原則
AIアシスタントは、ユーザーの要求や現在の実装に対し、より良いアプローチや代替案がある場合は積極的に提案する。

### 2. アイデアの保存 (Automatic Archiving)
議論の結果、**「今回は採用しないが、将来的に有用である」**または**「特定のユースケースでは正解である」**と判断されたアイデアについては、**ユーザーの指示を待たずに**以下の手順で保存する。

1.  **保存先**: `docs/management/IDEAS.md`
2.  **記載内容**:
    - **提案内容**: どのような機能を提案したか。
    - **採用しなかった理由**: なぜ今回は見送ったか（例：汎用性を優先するため、Base Kitとしては機能過多など）。
    - **コンテキスト**: どのような文脈で議論されたか。
3.  **タイミング**: 議論が収束し、実装方針が決定した直後。

これにより、貴重な議論の結果を埋もれさせることなく、将来のカスタマイズや機能拡張の際の「引き出し」として活用する。
