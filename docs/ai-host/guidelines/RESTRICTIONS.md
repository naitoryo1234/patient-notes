# 鉄の掟（禁止事項） (Restrictions)

以下のルールは絶対であり、ユーザーの明示的な許可なく破ってはなりません。

1.  **破壊的操作の禁止**:
    - `git reset --hard`, `git push -f` などの履歴改変コマンド。
    - ファイルやディレクトリの無断削除。
    - **ローカルDBファイル（sqlite.db等）の削除・上書き**。これが消えると患者データが飛びます。
    - これらが必要な場合は、必ずユーザーに確認を取り、「安全チェックモード」で手順を提示してから実行すること。

2.  **文字コード**:
    - すべてのテキストファイルは **UTF-8 (BOMなし)** で保存する。
    - 改行コードは **LF** (Unix style) を使用する。

3.  **推測による実装の禁止**:
    - 仕様が曖昧な場合、勝手に補完して実装を進めない。特に医療的なコンテキスト（カルテの記載ルールなど）は確認する。
    - 外部ライブラリを無闇に追加しない（バンドルサイズの肥大化防止）。

4.  **機密情報の漏洩禁止**:
    - APIキー、パスワード、個人情報などをコード内にハードコードしない。環境変数を使用する。
    - **ダミーデータであっても、実在の患者名や連絡先を使用しない**。

5.  **著作権の侵害禁止**:
    - 許可のない著作物（画像、コード例など）をプロジェクトに混入させない。

6.  **フェーズ外実装の厳禁 (Phase Boundary Protocol)**:
    - **合意なき機能拡張の禁止**: Phase 8以降の機能やロードマップ外の機能について、ユーザーとの明確な合意形成（Consensus）なしに実装に着手することを固く禁ずる。
    - **確認の徹底**: 実装フェーズに入る前に、必ず「何を、なぜ、どのように実装するか」を提案し、Goサインを得ること。
    - 「汎用的な土台」としての価値を損なわないため、推測や独断での機能追加は厳禁とする。
