# 📋 ダッシュボード運用レビューチェックリスト

**目的**: 汎用的な個人店舗（サロン、教室、クリニック等）の受付業務において、ダッシュボードが実務に耐えられるかを検証する。

**レビュー日**: 2025-12-11  
**対象バージョン**: v1.1.5

---

## ✅ レビュー項目

### 1. 予約状況の即座把握（朝の開店準備）

**シナリオ**: スタッフが朝、店舗を開ける際に「今日の予約状況」を一目で把握したい

- [ ] ダッシュボードを開いた瞬間に、本日の予約件数が分かるか
- [ ] 「担当未定」の予約が赤色で強調表示され、見落とさないか
- [ ] 「申し送り事項」がある場合、確実に目に入るか（赤色アラート）
- [ ] 最初の予約（次に来る顧客）がすぐに分かるか

---

### 2. 来店客のチェックイン（受付対応）

**シナリオ**: 予約していた顧客が来店。受付スタッフが素早く対応する

- [ ] 本日の予約カードから、該当顧客を素早く見つけられるか
- [ ] 「受付」ボタンをワンクリックで押せるか（ホバー時にボタン表示）
- [ ] チェックイン後、カードが視覚的に変化し「対応済み」が分かるか
- [ ] チェックイン操作がスムーズで、UI反応が早いか（楽観的更新）

---

### 3. 顧客検索（飛び込み客・問い合わせ対応）

**シナリオ**: 予約なしで来店した顧客、または電話問い合わせで顧客情報を探す

- [ ] 右カラムの検索フィールドがすぐに見つかるか
- [ ] 氏名・フリガナでの検索が問題なく動作するか
- [ ] 診察券番号（顧客ID）での検索が機能するか
- [ ] 検索結果から患者詳細へスムーズに遷移できるか
- [ ] 検索結果の「最終来店日」が表示され、判断材料になっているか

---

### 4. 予約変更・担当者調整（当日のオペレーション変更）

**シナリオ**: スタッフの急病や、顧客の時間変更希望に対応する

- [ ] 予約カードから編集モーダルへ素早くアクセスできるか
- [ ] 担当者の変更がプルダウンで簡単に行えるか
- [ ] 「未定」に戻すことができるか
- [ ] 時間変更時、重複があればエラーが表示されるか
- [ ] 変更後、ダッシュボードが即座に更新されるか

---

### 5. 要確認事項の管理（スタッフ間の伝言）

**シナリオ**: 前日の受付スタッフから朝番スタッフへの申し送り事項がある

- [ ] 右カラムの「要確認」タブに未割当予約が表示されるか
- [ ] 「要確認」タブに未確認の申し送り事項が表示されるか
- [ ] 申し送り事項をクリックして「確認済み」にできるか
- [ ] 確認済みにすると、アラート件数が減少するか
- [ ] 重要度（担当未定=黄色、申し送り=赤色）が視覚的に区別できるか

---

### 6. 自動更新とリアルタイム性

**シナリオ**: 複数のスタッフが同時にダッシュボードを開いている状態

- [ ] 1分ごとの自動更新が邪魔にならず動作しているか
- [ ] モーダル編集中は自動更新が停止し、編集が邪魔されないか
- [ ] 手動リフレッシュボタンが分かりやすい位置にあるか
- [ ] リフレッシュ時にスクロール位置がリセットされないか

---

### 7. 情報の優先順位と視認性

**シナリオ**: 予約が10件以上ある繁忙日

- [ ] 「これからの予約」と「完了・キャンセル」が分離されているか
- [ ] 重要なタグ（「要注意」等）が付いた顧客が埋もれていないか
- [ ] 時間経過で「間もなく来店」の予約が自動的に強調されるか（黄色背景）
- [ ] 過去の予約（完了分）が視覚的にトーンダウンしているか

---

### 8. 予約からカルテ記入への導線

**シナリオ**: サービス提供後、その場でカルテ（施術記録等）を記入したい

- [ ] 予約カードまたはモーダルから「カルテ作成」へ遷移できるか
- [ ] 遷移先で顧客情報が自動入力されているか
- [ ] 戻る導線が分かりやすいか

## 🔍 追加検証項目（汎用Base Kit観点）

- [ ] **他業種適用性**: 美容室、ヨガ教室、カウンセリングルームなど他業種でも違和感なく使えるか
- [ ] **文言の汎用性**: 「患者」「来院」などの医療用語が他業種で違和感を生んでいないか
- [ ] **カスタマイズ余地**: 将来的に文言を変数化・差し替えできる構造になっているか
