# ラベル管理・変数化の実装

## 概要
`src/config/labels.ts` を中心としたラベル管理システムを実装しました。これにより、クリニックやサロンなどの業態に合わせて、アプリ内の用語（患者、カルテ、施術者など）を一括して切り替えることが可能になりました。

## 実装内容
- **`src/config/labels.ts` への集約**: UI上のハードコード文字列をすべて変数化。
- **Variant System (`TERM_VARIANTS`)**: 業態ごとの用語セットを定義。現在は `CLINIC` がデフォルト。
- **コンポーネント適用**: 予約管理、スタッフ管理、ダッシュボード等、主要画面への適用完了。
- **スタッフ役割の統合**: スタッフの役割（院長、施術者など）もバリアントに含めることで、業態変更時の柔軟性を確保（ユーザー要望による対応）。

## 今後の運用
- 用語を変更したい場合は、`labels.ts` 内の定義を変更するだけでアプリ全体に反映されます。
- 新しい役割（例: 受付、マネージャー）を追加する際は、`STAFF` セクションおよび `TERM_VARIANTS` の拡充を行ってください。
