# 商品パッケージ戦略 (Product Packaging Strategy)

## 📦 コンセプト: "Start Small, Expand as Needed"

本製品は、不必要な機能で肥大化した業務システムへのアンチテーゼとして設計されています。
**「個人事業主にとって必要十分なコア機能 (Base Kit)」** を提供し、事業の成長やニーズに合わせて **「必要な機能だけを追加 (Plugin)」** するモデルを採用します。

---

## 🛠️ Base Kit (Standard Edition)

**"これだけで開業できる"**
単なる「お試し版」ではなく、**それ単体でプロフェッショナルな業務を完結できる**有料パッケージです。

### 提供価値
- **オフライン・ファースト**: ネットがなくても動く（Electron版）。
- **データ主権**: 顧客データは自分のPC内にあり、誰にも覗かれない安心感。
- **超高速動作**: クラウドの遅延がないサクサクとした操作感。

### コア機能 (Included)
| カテゴリ | 機能概要 |
| :--- | :--- |
| **台帳管理** | 顧客情報、対応履歴、属性タグ付け (Generic Labels) |
| **予約管理** | ドラッグ＆ドロップ対応の予約カレンダー、重複チェック |
| **電子カルテ** | シンプルなテキスト入力、AI音声入力用フォーマット |
| **データ保全** | ワンクリックでの完全バックアップ (JSON/SQLite) |
| **基本設定** | 医院/サロン情報の編集、スタッフ管理、施術メニュー設定 |

## 🧩 Plugin Ecosystem (Value Add)

**"かゆいところに手が届く"**
Base Kitの価値をさらに高め、特定のニーズに応えるための**追加購入型（またはサブスクリプション型）拡張機能**です。

### Phase 1 Plugins (Available Now / Coming Soon)

#### 1. 📷 Image Attachment Plugin
*   **概要**: 施術前後の写真や、紙の紹介状などをカルテに添付する機能。
*   **ターゲット**: 美容鍼灸、整体、エステなど「変化」を記録したい業種。
*   **実装状況**: 実装済み (v1.3.2) - 設定でON/OFF可能。

#### 2. 🔍 Advanced Search Plugin
*   **概要**: 「50代 女性 腰痛」のような複合条件での患者検索や、来院頻度分析。
*   **ターゲット**: リピート率改善に取り組みたい経営者。
*   **実装状況**: 計画中。

### Future Plugins (High Value)

#### 3. 💬 LINE Connect
*   **概要**: 予約確認やサンキューメッセージをLINE公式アカウントから自動送信。
*   **収益モデル**: 月額課金 (SaaS版のみ、またはAPI連携)。

#### 4. 🤖 AI Assistant (Summary & Suggest)
*   **概要**: 過去のカルテから「前回の施術内容」を要約したり、次回提案をレコメンド。
*   **ターゲット**: 提案力を強化したい施術者。

## 🚀 配布・ライセンスモデル

### A. Customer Notebook (Local / Electron)
*   **形態**: 買い切りライセンス + 年間サポート(任意)
*   **Plugin**: ライセンスキー入力による機能アンロック方式（予定）。
*   **ターゲット**: データの自己管理を重視する個人院、ネット環境が不安定な現場。

### B. Customer Notebook Cloud (SaaS / Vercel)
*   **形態**: 月額サブスクリプション (SaaS)。
*   **Plugin**: プランアップグレードによる機能開放。
*   **ターゲット**: 複数スタッフで同時アクセスしたい医院、PC管理を任せたい層。

## 📝 技術的実現性への示唆

この戦略を実現するため、コードベースは以下の特性を維持する必要があります。

1.  **フィーチャーフラグ駆動**: 全てのプラグイン機能は `FeaturesProvider` を通じてON/OFF制御可能であること。
2.  **設定の永続化**: プラグインの有効化状態は `SystemSetting` テーブル（DB）に保存され、再起動後も維持されること。
3.  **モジュラー構造**: プラグインコードは `src/plugins/` 配下に集約し、Base Kitへの影響を最小限にすること。
