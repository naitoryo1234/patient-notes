# 手動確認リスト (Manual Verification List)

Level 3テスト（全量確認）において、Browser Automationでは完全に検証しきれない、または「手触り」を確認すべき項目のリストです。
次回セッション開始前、またはお時間のある時にご確認をお願いします。

---

## 🎨 UI/UXの手触り確認

### 1. カレンダー操作 (C-06)

- **対象画面**: `/appointments` (月/週ビュー)
- **手順**:
  1. 既存の予約カードをドラッグして、別の日付や時間に移動させる
  2. ドロップ時の挙動がスムーズか確認する
  3. 移動後、更新が即座に反映されるか確認する
- **確認ポイント**: 操作感の「重さ」や、ドロップ判定のシビアさがないか
- **ステータス**: [ ] 未確認

---

### 2. 長文レイアウトの崩れ (PT-12)

- **対象画面**: `/patients/[id]` (詳細ページ)
- **手順**:
  1. 生成されたスクリーンショット `long_name_layout.png` を確認する
  2. または、テストデータとして登録された「あいうえおかきくけこ...」という患者を実際に表示する
- **確認ポイント**:
  - 名前がヘッダーからあふれていないか
  - パンくずリストやサイドバーを圧迫していないか
- **ステータス**: [ ] 未確認

---

## ⚙️ 機能の動作確認

### 3. バックアップファイルの検証 (BK-01)

- **対象画面**: `/settings` (データ管理)
- **手順**:
  1. 「バックアップを保存」ボタンをクリックし、実際に `.db` ファイルをダウンロードする
  2. ダウンロードされたファイルサイズが **0バイトでない**ことを確認する
  3. (可能であれば) ファイル名の日時が現在日時と一致しているか確認する
- **ステータス**: [ ] 未確認

---

### 4. 認証OFF時の挙動 (ENV-03)

- **対象画面**: アプリ起動時
- **手順**:
  1. シークレットウィンドウなどで `http://localhost:3000` にアクセスする
  2. ログイン画面が表示されず、即座にダッシュボードが表示されることを確認する
- **ステータス**: [ ] 未確認
