# テスト実行ガイド (Test Execution Strategy)

128件のテスト項目を効率的かつ効果的に実行するためのガイドラインです。
一度に全て実行するのではなく、**優先度に基づいた3段階のセッション**に分けて実施することを推奨します。

---

## 🏗️ Level 1: Core Flow (生存確認)
**目的**: アプリケーションの基本機能が動作し、業務が止まらないことを確認する。
**頻度**: デプロイ前、大きな機能追加後
**所要時間**: 約10分

1. **予約作成 (AP-01, AP-06)**
   - ダッシュボードから1件予約を作成する。
2. **カレンダー確認 (C-01)**
   - 予約がカレンダーに表示されているか確認する。
3. **来院チェックイン (DA-06)**
   - ダッシュボードで「受付」ボタンを押す。
4. **カルテ作成 (RC-01, RC-04)**
   - 患者詳細からカルテを1件作成し保存する。
5. **会計/完了 (DA-07)**
   - ダッシュボードで「完了」ボタンを押す。

---

## 🏃 Level 2: Standard Operations (業務網羅)
**目的**: 日常業務で発生する各種変更操作や、複数人運用での挙動を確認する。
**頻度**: 定期テスト、UI変更時
**所要時間**: 約20分

1. **イレギュラー予約対応**
   - 予約変更 (AP-08)、キャンセル (AP-11)
   - 重複チェックの確認 (AP-14, AP-16)
   - ドラッグ＆ドロップ移動 (C-06)
2. **患者管理**
   - 新規患者登録 (PT-01)
   - 検索機能の動作 (PS-01, PT-13)
3. **申し送り・連携**
   - 申し送りの入力と解決 (AP-18, AP-19)
   - ダッシュボードのアラート確認 (DA-04)

---

## 🔍 Level 3: Full Verification (全量確認)
**目的**: 設定、管理機能、エッジケースを含めたシステムの完全性を保証する。
**頻度**: メジャーリリース前、大規模改修時
**所要時間**: 約30-40分

1. **設定・管理**
   - スタッフの追加・無効化 (ST-01〜06)
   - プラグイン設定切替 (PL-01)
   - バックアップ手動ダウンロード (BK-01)
2. **エッジケース**
   - バリデーションエラー確認 (PT-02)
   - ページネーション (AP-25)
   - 長い名前の表示崩れ確認 (PT-12)
   - 認証機能の動作 (AU-01〜07)
