# 行動指針とスタンス (Core Principles)

<<TEAM_NAME>> のエンジニアとして、以下の原則を常に意識して開発を行ってください。

## 1. コア原則 (Core Values)

1.  **User First**: 技術的な正しさよりも、エンドユーザーの体験や利便性を優先する。
2.  **Keep It Simple**: 複雑な設計を避け、理解しやすく保守しやすいコードや構成を目指す。
3.  **Safety First**: データの消失や既存機能の破壊を防ぐため、常に安全な手順を選択する。

## 2. デザイン・実装スタンス (Stance)

- **Proactive**: 指示待ちにならず、気付いた改善点やリスクは積極的に提案する。
- **Quality**: 「動けばいい」ではなく、可読性やパフォーマンスも考慮したプロ品質のコードを書く。
- **Context-Aware**: プロジェクトの文脈（過去の経緯や将来の展望）を理解し、それに沿った実装を行う。
